# S — Принцип единственной ответственности
Каждый класс выполняет только одну задачу:
+ Animal - абстрактный класс животных
+ Thing - абстрактный класс предметов (инвентарь)
+ Predator, Herbo - классы типов животных
+ Monkey, Wolf, ..., Table, Computer - конкретный предмет или конкретное животное
+ Zoo управляет зоопарком
+ VeterinaryClinic проверяет здоровье

# L — Принцип подстановки Лисков
Herbo наследуется от Animal
Класс-наследник не изменяют поведение базового класса, а расширяет (дополнительное поле kind и соответствующую ему логику).
Все классы-наследники работают корректно при подстановке в базовый класс.

# I — Принцип разделения интерфейса
Все интерфейсы узконаправлены, среди нет интерфейса "общего назначения":
+ IAlive - отвечает за все живое (Food, Health)
+ IHealthCheck - отвечает за проверку здоровья 
+ IInventory - отвечает за инвентарь (Number)
+ IKind - отвечает за уровень доброты (KindLevel)

# D — Принцип инверсии зависимостей
Класс zoo зависит от интерфейса IHealthChecker, а значит можно будет в будущем использовать любую организацию для проверки здоровья.

# O — Принцип открытости/закрытости
Объекты открыты для расширений и закрыты для модификации
+ Можно добавить новых животных
+ Можно добавить новые предметы
+ Можно добавить новое учреждение для проверки здоровья
